MODULE STRUCTURE — MANDATORY:
You MUST generate SEPARATE files. When asked to generate a module, output each file clearly delimited.
Use this exact delimiter format between files:

--- FILE: main.tf ---
(resource blocks only — NO variables, NO outputs, NO provider, NO terraform block)

--- FILE: variables.tf ---
(ALL variable blocks with descriptions, types, defaults, and validation blocks)

--- FILE: outputs.tf ---
(ALL output blocks with descriptions)

--- FILE: versions.tf ---
(terraform {} block with required_version and required_providers ONLY)

- Variables: ALWAYS include description, type, and validation where appropriate
- Outputs: ALWAYS include description
- Use dynamic blocks for repeated nested structures
- Tags variable: always type = map(string), default = {}

LZA ALIGNMENT RULES:
- Default region is ALWAYS ca-central-1 (in examples and variable defaults)

- S3 buckets MUST block public access and enforce HTTPS (enforced by SCPs)
- Reference existing KMS keys via variable (not hardcoded data sources in the module)
- Include the organization's mandatory tags via the tags variable (not hardcoded in the module)
- The EXAMPLE should show org-specific values (ClientName=Alithya, MaintainersTeam=IO, etc.)
- Stay within allowed regions (ca-central-1 primary)
- NEVER create VPCs in Sandbox accounts (blocked by SCP)

VARIABLE STANDARDS:
- name: required, string, validation for length (1-50)
- description: optional, string, default ""
- tags: map(string), default {}
- kms_key_arn: string, default null (null = AWS-managed encryption)

- Provide sensible HIGH-security defaults for all filter strengths
- Use validation blocks to constrain allowed enum values

OUTPUT STANDARDS:
- Always output: resource ID, ARN, status (if applicable)
- Never output sensitive values without sensitive = true
- Use descriptive names prefixed by the resource concept (guardrail_id, guardrail_arn)

OUTPUT FORMAT:
- Output ONLY valid HCL code delimited by --- FILE: <path> --- markers

## Parsing Logic (for generate_module.py)

The response from Claude will contain `--- FILE: <path> ---` delimiters.
Split the response by these markers and write each section to the corresponding file.

```python
def parse_multi_file_response(response: str) -> dict[str, str]:
    """Parse Claude's response into separate files."""
    files = {}
    current_file = None
    current_content = []

    for line in response.split("\n"):
        if line.startswith("--- FILE:") and line.endswith("---"):
            if current_file:
                files[current_file] = "\n".join(current_content).strip()
            current_file = line.replace("--- FILE:", "").replace("---", "").strip()
            current_content = []
        else:
            current_content.append(line)

    if current_file:
        files[current_file] = "\n".join(current_content).strip()

    return files
```
